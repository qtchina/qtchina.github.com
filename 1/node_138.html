发现一个开发很活跃的ssh库项目 ： libssh2
发布： 2008-05-28 20:03

项目主页：http://sourceforge.net/projects/libssh2/

Developers : 14

Development Status : 4 - Beta

Intended Audience : Developers

License : BSD License

Activity Percentile (last week) : 99.29

CVS Repository : (364 commits, 7,475 reads)

虽然开发很活跃，可现在仍然是beta版的，主要是ssh协议太复杂了。这个项目的文档还比较好：http://www.libssh2.org/wiki/index.php/Main_Page ， 比openssh 多。openssh 虽然做的非常好，要单独摘出来哪一块不太容易，甚至有些代码还很诡异，代码耦合度相当高，还没有文档。

不过有这样一个库对nullfxp以后实现同时连接多台服务器，服务器与服务器之间的直接交互，甚至多线线程处理上下传文件，都能有很大帮助。

另外它还支持多平台，其中包括Windows,对nullfxp向其他系统移植比现在这种方式要简单。

在nullfxp 1.0 之前，实现启动一个nullfxp进程对远程sftp服务器交互的功能，包括上下传文件，远程主机上的文件系统管理，如创建目录，删除目录，目录或者文件改名，目录或者文件的移动等。并不断加强稳定性。

进入1.0之后，应该libssh2库也有所进展，移植到这个库上来。

提供一个nullfxp实例同时连接到多台sftp服务器的功能，并实现1.0之前各功能。

同时移植到其他平台。

2.0 ，加入ftp协议支持

总的看来，1.0阶段是比较难的一步。

今天用libssh2 写了一段示例代码，感觉可控性比用OpenSSH分离出来的SSH协议库好多了。正准备将nullfxp移植到这个库上来，冲刺1.0版的 nullfxp 了。回头把使用libssh2的使用经验写出来。
