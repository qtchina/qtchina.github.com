1.1. VideoLAN 工程是什么 ?

1.1.1.概述

VideoLAN 是一个完整的视频流解决方案,由 Ecole Centrale Paris 的学生和世界各地的开发者开发, 基于GNU General Public License (GPL). VideoLAN被设计成在宽带网络上播放视频流.

VideoLAN解决方案包括:

  • VLS (VideoLAN 服务器)能传送 MPEG-1, MPEG-2 and MPEG-4 流文件, DVDs,数字卫星频道, 数字电视频道与基本网络单播和多播的即时流。

  • VLC (initially VideoLAN 客户端), 可用来作基于网络单播和多播的MPEG-1, MPEG-2 and MPEG-4流媒体文件，DVDs,即时视频服务器 ; 或者用作多用户系统上的接收，解码与播放MPEG流媒体的客户端。

1.1.2. VideoLAN 软件

1.1.2.1. VLC

VLC 可工作于许多平台 : Linux, Windows, Mac OS X, BeOS, *BSD, Solaris, Familiar Linux, Yopy/Linupy and QNX. 它能读:

  • 硬盘上的MPEG-1, MPEG-2 and MPEG-4 / DivX 文件k, CD-ROM 文件 ...

  • DVDs 和 VCDs,

  • from a satellite card (DVB-S),

  • 网络上的VLS或VLC 输出流:MPEG-1, MPEG-2 和 MPEG-4.

VLC 也可用作从can also be used as a server to stream :

  • MPEG-1, MPEG-2 和 MPEG-4 / DivX 文件,

  • DVDs,

  • MPEG 编码卡,

到 :

  • 一台机器 (例如到一个 IP地址) : 这叫单播,

  • 动态的客户可自由加入和离开的机器组 (例如到一个多播IP地址) :这叫多播。

支持 IPv4 或 IPv6 .

要获得所有可能平台支持完整列表，请看 VLC features page.

VLC 不适用在Mac OS 9平台, 并用可能永不会.

1.1.2.2. VLS

VLS 能编码符合下面条件的流 :从

  • 硬盘上的MPEG-1, MPEG-2 或 MPEG-4 和 CD上的文件,

  • DVD驱动器上的DVD 或者硬盘上的拷贝,

  • a satellite card (DVB-S) or a digital terrestial television card (DVB-T),

  • MPEG 编码卡 ;

到 :

  • 一台机器 (例如到一个IP 地址) : 这叫单播,

  • 动态的客户可自由加入和离开的机器组 (例如到一个多播IP地址) :这叫多播。

支持 IPv4 或 IPv6.

在网络上一台32MB和 Pentium 100 MHz的机器就足够了。当要播放大量硬盘上的视频流时，实际的限制不是处理器，而是硬盘和刚强连接。

VLS 可工作于 Linux 和 Windows.要获得平台支持的完整列表，请看 streaming features page.

1.1.2.3. Mini-SAP-server

你可以添加一个基于SAP/SAD标准的频道信息服务到VideoLAN解决方案中。 mini-SAP-server 可以在IPv4或IPv6网络上传递多播节目的预报，并且VLCs可接收预报并自动加入到播放列表中.

mini-SAP-server 工作于 Linux 和 Mac OS X.

1.2. 如何使用 VideoLAN ?

1.2.1.文档

用户文档是由下面4部分组成:

  • VideoLAN Quickstart. 这一文档给你一个快速的VLC概述，VLC的流输出，视频演示解决方案和频道信息服务系统 。

  • VideoLAN HOWTO. 这一文档是完整的VideoLAN视频流解决方案的指导。

  • VLC user guide. 这一文档是完整的VLC指导。

  • VLS user guide. 这一文档是完整的VLS指导。

  • VideoLAN FAQ. 这一文档包括有关VideoLAN的常见问题。

这些文档的最新版本可以在 documentation page找到。

你还可看看VideoLAN Wiki. 这是一个每个人都可更改的网页。我们用它来做非"官方"的所有文档：每个系统的使用技巧，图形接口等。

1.2.2.使用支持

如果你在使用videoLAN时有问题,并且你在这个文档中没有找到问题的解答，请看 online archive of the mailing-lists. 这有两个为用户准备的邮箱:

  • vlc@videolan.org 为 VLC 的问题,

  • streaming@videolan.org 为 VLS 的问题, mini-SAP-server 网络 。

如果你想订阅或者解除订阅上面的邮件，请到 mailing-list page.

你还可在IRC:irc.freenode.net, channel #videolan上与VideoLAN用户和开发都交流。

如果你发现了bug, 请看下面的介绍 bug reporting page.

1.3. 命令行用法

  • VLC 有许多不同的图形接口 ，它们为每个操作系统的指导作协调的组织。为第一个图形界面写文档实在太长，并且一些特性林适用于命令行，但在大多数时候可以很容易的猜到如何用图形接口。

  • VLS 有命令行和telnet 接口，但没有图形接口。

这篇文档所示的所有命令要运行在终端上。
